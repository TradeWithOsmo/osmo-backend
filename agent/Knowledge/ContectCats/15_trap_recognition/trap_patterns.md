# False Signal & Trap Recognition

## Core Principle
*If it looks too obvious, the market is setting a trap.*

## Common Trap Patterns

| Trap Type | Description | Warning Signs |
|-----------|-------------|---------------|
| **Fake Horizontal Breakout** | First break of major S/R fails | Low volume, against HTF trend |
| **Trendline Break Without Volume** | Line breaks but no follow-through | Volume < average |
| **Long Wick at Liquidity Zone** | Wick sweeps stops, then reverses | Wick at round number, previous high/low |
| **Perfect Setup Syndrome** | Too many confluences, everyone sees it | "Obvious" entries often trapped |
| **Weekend Gap Trap** | Gap fills but continues against you | Fading gaps on Monday risky |

## Trap Detection Rules

- **Volume Check:** No volume on break = suspicious
- **HTF Alignment:** Break against HTF = likely trap
- **Obvious Level:** If you see it easily, so do institutions (they fade it)
- **First Attempt:** First break of major level often fails

## Trap Memory: First Break Usually Fails

```
💾 **TRAP MEMORY**

Historical observation:
- 70% of first breaks of well-known resistance fail
- Smart money uses first break to fill shorts
- True breakout usually comes on 2nd or 3rd attempt

Current application:
- BTC just broke $100,000 for the first time
- High probability this is a trap
- Recommend: Wait for retest and hold, or second break attempt
```

## Agent Warning Format

"⚠️ This is the first break of this major level. Historical pattern suggests first breaks often fail. Consider waiting for retest confirmation."

## Pre-Entry Trap Check

```
🚨 **TRAP CHECK**

Before entering this setup, verify:
1. Is this the first break of the level? ⚠️ (often fails)
2. Is volume below average? ⚠️ (suspicious)
3. Is this against HTF trend? ⚠️ (counter-trend trap)
4. Is this an "obvious" setup everyone sees? ⚠️ (crowded)
5. Did a long wick just sweep liquidity? ⚠️ (stop hunt)

Trap flags: [X/5]
If ≥3 flags → Consider skipping or reduced size.
```

## Trap Prevention Workflow

### Before Any Entry
1. Run trap checklist
2. Count warning flags
3. If ≥ 3 flags: Skip or reduce size to 30%
4. If 1-2 flags: Normal size but tight SL
5. If 0 flags: Full size OK

### After Failed Trap
- Document pattern
- Add to memory
- Look for counter-play opportunity
